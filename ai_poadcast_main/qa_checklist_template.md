# QA Checklist 模板

> 目的：为每一期播客提供可复用的质检框架，确保要点卡片、脚本与音频在上线前经过一致的检查流程。使用时复制本页到 `ai_poadcast_main/stage3_reviews/YYYY-MM-DD/episode_<date>_review.md` 或独立 QA 文档，并勾选完成项、补充发现的问题。

---

- **期次 / 日期：** `2025-10-29`
- **执行人：** `QA 审核人姓名`

  - `...`

## 阶段 2（要点卡片）质检

| 检查项 | 说明 | 结果 | 备注 |
|--------|------|------|------|
| [ ] 事实匹配 | 所有数字、时间、地点与英文原文一致 |  |  |
| [ ] 术语标准化 | Prompt 2 输出已套用最新术语库 |  |  |
| [ ] 数字/日期校验 | Prompt 3 检查通过，单位、汇率清晰 |  |  |
| [ ] 结构完整 | 5W1H、关键数据、原文引用完整无缺项 |  |  |
| [ ] 文件命名正确 | `*_card_final.md` 存在，过时稿已归档 |  |  |

> 若有未通过项，请在“备注”中说明原因及修正举措，并在完成后重新勾选。

---

## 阶段 3（脚本）质检

| 检查项 | 说明 | 结果 | 备注 |
|--------|------|------|------|
| [ ] 来源标注 | 每条新闻均注明来源机构/日期 |  |  |
| [ ] 事实无偏差 | 脚本全部事实可在卡片查证 |  |  |
| [ ] 风险用语 | 无 “保证”“唯一”“免费” 等高风险表述 |  |  |
| [ ] 免责声明 | 签证/移民/考试建议附“以官网为准”等提示 |  |  |
| [ ] 易读性 | 朗读顺畅，无 >30 字长句、拗口表达 |  |  |
| [ ] 版本归档 | `episode_<date>_v1/v2/...`、`_final` 文件存在 |  |  |

补充说明：
- **朗读测试**：请录制或现场朗读一次，发现卡顿句子后在备注中记录调整情况。
- **反馈闭环**：重大修改（如新增事实、删减段落）需同步回 `Master action plan` 或 Stage 2 终稿。

---

## 阶段 4（音频）质检

| 检查项 | 说明 | 结果 | 备注 |
|--------|------|------|------|
| [ ] 声音质量 | 无噪音、爆音、断句或明显合成瑕疵 |  |  |
| [ ] 语速语调 | 与脚本节奏一致，停顿自然 |  |  |
| [ ] 文案一致 | 音频内容与 `episode_<date>_final.md` 完全一致 |  |  |
| [ ] 渠道格式 | 输出格式符合播出平台要求（如 44.1kHz/16bit） |  |  |
| [ ] 文件命名 | `audio_exports/YYYY/episode_<date>_<voice>.ext` |  |  |
| [ ] 审核记录 | 本模板或 `stage3_reviews/..._review.md` 已填写并归档 |  |  |

> 如需修订音频，请记录重新合成时间、调整内容及最终采用的文件名。
